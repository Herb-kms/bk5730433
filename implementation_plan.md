# 제품 상세/판매 페이지 관리자 기능 확장 계획서 (추가 요구사항 반영)

제품 등록/수정 모달에 두 가지 노출 프리뷰를 추가하고, 제품 상세 페이지에서 직접 수정할 수 있는 권한과 항목을 넓히기 위한 설계 계획서입니다.

---

## 1. 주요 요구사항 및 구현 설계

### ① 이중 노출 프리뷰 기능 제공 (탭 전환 방식)
제품 등록/수정 모달(관리자 페이지 & 제품 상세페이지 모달)의 우측 프리뷰 영역을 탭 형식으로 개편합니다.
- **노출 프리뷰 1번 (목록 카드)**: 임대제품/판매제품 목록 화면에 노출되는 기존 `prod-card` 디자인의 프리뷰.
- **노출 프리뷰 2번 (상세페이지)**: `product-detail.html` 화면 구성의 핵심 정보(브레드크럼, 배지, 제품명, 설명, 견적 시뮬레이터 카드, 상세 사양 테이블)를 미니어처 레이아웃으로 모방한 새로운 프리뷰.
- **실시간 프리뷰 동기화**: 관리자가 폼에 입력할 때마다 카드 프리뷰와 상세페이지 프리뷰의 제품명, 가격, 사양, 설명, 배지가 실시간으로 연동되어 갱신됩니다.

### ② 모바일 헤더 투명화 및 가로 스냅 슬라이더 구현
- **[모바일 헤더 투명도 & 바탕 일치화]**: 모바일 화면에서도 헤더(`.hero-nav-bar-fluid`)가 하얗게 분리되지 않고, 오리지널 푸른 동적 그라데이션 배경 위에 투명하게 녹아들도록 스타일을 튜닝합니다.
- **[자석식 스크롤 스냅(Scroll-Snap) 도입]**: 사용자가 모바일 카드를 손가락으로 넘길 때, 카드가 화면 정가운데에 자석처럼 멈추는 CSS 기술을 탑재하여 네이티브 앱 같은 조작감을 선사합니다.

### ③ 제품 상세페이지 내 편집 권한 및 영역 확장
- 상세페이지(`product-detail.html`)의 수정 폼에서 기존 이미지 표시 버그를 개선하여, 모달 오픈 시 현재 기기의 기존 이미지가 파일 업로드 박스 내에 기본 프리뷰로 출력되도록 수정합니다.
- 상세페이지 수정 모달 내부에서도 목록 카드와 상세페이지 레이아웃의 실시간 2개 프리뷰를 모두 똑같이 제공합니다.
- 데이터베이스 갱신 시 `FormData` 전송 로직이 모든 핵심 컬럼을 완벽하게 덮어쓰도록 권한 및 처리 범위를 넓힙니다.

## 3. 검증 계획

### 수동 검증 시나리오
1. 관리자로 로그인 후 상세페이지 또는 관리자 페이지에서 수정 모달 진입.
2. 우측 상단의 **[노출 프리뷰 1 (목록 카드)]**와 **[노출 프리뷰 2 (상세페이지)]** 탭 클릭 시 프리뷰 영역이 알맞게 스위칭되는지 확인.
3. 폼 양식을 편집(예: 가격 변경, 제품명 변경, 스펙 변경)할 때 1번 카드와 2번 상세페이지 프리뷰가 각각 실시간 동적 연동되는지 검증.
4. 새로운 제품 사진을 업로드해보고 프리뷰 및 수정 저장 기능 검증.
